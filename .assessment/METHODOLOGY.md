# Audit methodology

This repo's engineering audit was produced by the **codebase-audit** rubric (version 1.0.0) — a
standardized, deterministic scoring of a repository's **engineering quality as it currently exists**
(not product completeness or feature coverage). The same rubric was applied identically to every arm of
the comparison, so the scores are directly comparable.

## How the score is computed

Nine categories are each scored **0–5**, multiplied by a fixed weight, and summed to a 0–100 overall:

| Category | Weight |
|---|:--:|
| Architectural coherence | 15 |
| Implementation quality | 15 |
| Operational readiness | 15 |
| Type safety & contracts | 10 |
| Test strategy | 10 |
| Scalability & extensibility | 10 |
| Security & failure handling | 10 |
| Engineering discipline | 10 |
| Technical debt & risk | 5 |

`weighted = round(score / 5 × weight)` per category; `overall = round(sum)`.

**Bands:** 90–100 exceptional · 75–89 strong_foundation · 60–74 functional_with_weakness ·
40–59 fragile · 0–39 critically_immature.

## Guardrails that keep it honest

- **Score caps (hard ceilings):** implementation quality is capped at 3 if the build/typecheck fails with
  no documented reason; operational readiness at 2 if there's no runnable/deploy path; test strategy at 3
  if no tests exist — and a disqualifier holds test strategy at 3 unless tests actually run in CI (which is
  why every arm in this comparison scored 3 there).
- **Evidence required:** every strength, risk, and score driver must carry a citation — a `file:line`, a
  `command:<name>#exit=<n>`, or a `structural:<fact>` marker — enforced by a regex at validation time.
  Prose-only claims are rejected. Each category has a minimum evidence count (2–3).
- **Confidence** is recorded per category (high / medium / low) and surfaced in the report.
- **Deterministic:** same repo state + same rubric version → same score. A validator checks the citation
  grammar, evidence minimums, cap math, and weighted-sum consistency before the report is emitted.

## What to read

- [`report.md`](report.md) — the full human-readable report (scores, strengths, risks, with citations).
- [`report.json`](report.json) — machine-readable equivalent.
- [`categories/`](categories) — one file per category: its score, raw (pre-cap) score, confidence, and
  every cited piece of evidence behind it.
- [`rubric.yml`](rubric.yml) — the canonical rubric: weights, score caps, bands, citation grammar.

The audit is generated by the `codebase-audit` skill; it scores engineering quality and explicitly does
**not** reward or penalize product/feature completeness.
